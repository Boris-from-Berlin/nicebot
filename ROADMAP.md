# NiceBot Roadmap

This is where the project is going. Nothing here is fixed. The community shapes the direction.

- [x] Define the five axioms
- [x] Write the Manifesto
- [x] Build the landing page

- [ ] Launch the GitHub repo publicly
- [ ] Publish the documentary film

## Phase 1 — First working agent (Q3 2025)
- [ ] PrivacyGuard — functional privacy risk scanner
- [ ] ThreatRadar — basic cyberattack pattern detection
- [ ] TruthLayer — disinformation signal detection
- [ ] EthicsAdvisor — Codex-based decision questioning
- [ ] PersonalValues — configurable user value stack
- [ ] API-agnostic: Claude, OpenAI, Gemini support
- [ ] CLI interface
- [ ] Basic test coverage

## Phase 2 — Community and integrations (Q4 2025)
- [ ] Browser extension (Chrome / Firefox)
- [ ] Telegram bot integration
- [ ] Obsidian plugin for the knowledge graph
- [ ] Community survey results published
- [ ] First Codex revision based on community input
- [ ] 500+ GitHub stars
- [ ] Documentation in 3+ languages

## Phase 3 — Multi-agent system (2026)
- [ ] Full sub-agent coordination system
- [ ] NiceBot instances that consult each other
- [ ] Persistent memory with user consent
- [ ] Open API for third-party integrations
- [ ] Funding structure for continued development

- How do we prevent NiceBot from being captured by a single ideology?
